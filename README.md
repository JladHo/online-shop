# online-shop
Тут я изучаю фуллстек в среде js.  
Стек технологий: PostgreSQL + React js + Express + Node.js

В ходе этой практики у меня уже имеется некоторый опыт с JS и React, а также немного практиковался в разработке REST
на Java SpringBoot и на Golang с помощью Gin.

Цель этой практики - тщательнее изучить один конкретный стек технологий для разработки Backend и Frontend, чтобы
прокачать свои навыки, изучить новые инструменты, и, конечно же, не бегать со стека на стек в бесконечном поиске чего-то))
Лучше хорошо изучить один стек, чем по чуть-чуть много других.  

В будущем, конечно же, буду еще и другие стеки изучать. Этот взял за стартовую точку и буду изучать его как можно глубже,
чтобы понимать основы и принципы Fullstack разработки.

Этот проект делается на основе видео с канала UlbiTV.  
Ссылка на видео: https://www.youtube.com/watch?v=H2GCkRF9eko&list=PL6DxKON1uLOFJ5_dDcX7G1osKnsBlCaaT

# (UPD 12.11.2025)
Подумал и решил, что здесь буду еще оставлять комменты по поводу того, что делал и с чем были проблемы для того, чтобы
сохранить хронологию обучения.

Что я уже сделал к этому моменту:
-
- установил зависимости
- сделал функцию для запуска сервера
- добавил файл конфигурации
- создал модели
- добавил роуты
- добавил контроллеры для роутов
- добавил класс для обработки ошибок
- создал свой первый миддлвейр для ошибок
- начал реализовывать добавление девайсов с сохранением картинок в static

С чем возникли трудности:
-
- Трудности возникли с вызовом кастомных ошибок из ApiError. Это было из-за того, что в классе ApiError все 
ошибки вызываются через статические методы (т.е. через класс, а не через экземпляр класса), а импортировал из модуля 
я не сам класс, а экземпляр этого класса: `module.exports = new ApiError()`.  
**Решение**: `module.exports = new ApiError()` заменить на `module.exports = ApiError`.
